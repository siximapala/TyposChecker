# Поиск опечаток в тексте при помощи триграмм

2 shell скрипта для поиска возможных опечаток в русскоязычных текстах с использованием триграммного анализа.

Идея в том, чтобы определить все триграммы, которые встречаются в корректных русских словах, а затем находить в произвольном тексте триграммы, отсутствующие в этом наборе. Такие триграммы с высокой вероятностью указывают на опечатки или некорректные фрагменты текста.

Проект полностью на стандартных UNIX-утилитах.

## Получение валидных триграмм

Для построения словаря корректных триграмм используется словарь русских слов из репозитория  
https://github.com/danakt/russian-words/blob/master/russian.txt

Исходный файл находится в кодировке Windows-1251, поэтому перед использованием его необходимо преобразовать в UTF-8:

```
iconv -f WINDOWS-1251 -t UTF-8 russian.txt > russian.utf-8
```

После этого можно запустить скрипт генерации триграмм:

```
./missing_trigrams.sh russian.utf-8
```

В результате будет создан файл `trigrams_missing.txt`, содержащий список триграмм, которые не встречаются ни в одном корректном русском слове. Именно этот файл используется далее для поиска опечаток.

## Поиск опечаток в тексте

Для анализа текста используется скрипт `find_typos.sh`. Он принимает файл с невалидными триграммами и текст для проверки, разбивает текст на слова и триграммы и ищет совпадения с невалидным списком.

Пример запуска:

```
./find_typos.sh trigrams_missing.txt text_utf8.txt
```

Все найденные триграммы, отсутствующие в словаре корректных, сохраняются в отдельный файл и могут рассматриваться как потенциальные опечатки.

## Кодировки

Все входные файлы должны быть в кодировке UTF-8. При использовании текстов из внешних источников рекомендуется предварительно проверить и при необходимости преобразовать кодировку. Для корректной работы скриптов лучше использовать файл `text_utf8.txt` или вручную вставлять текст в UTF-8.

## Примечания

Для проверки можно использовать любые русскоязычные тексты большого объёма. Чем больше словарь исходных слов, тем точнее будет результат. Качество поиска напрямую зависит от полноты набора валидных триграмм.
